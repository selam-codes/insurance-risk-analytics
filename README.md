# insurance-risk-analytics

## Project Overview

Insurance risk analytics project for AlphaCare Insurance Solutions (ACIS). The project analyzes historical insurance data to optimize marketing strategies and identify low-risk customer segments for premium adjustments.

---

## Data Version Control (DVC)

This project uses [DVC](https://dvc.org/) to track and version large data files, ensuring reproducibility and auditability of the data pipeline.

### Prerequisites

```bash
pip install -r requirements.txt
# or just: pip install dvc
```

### DVC Setup

DVC is initialized with a **local remote storage** at `/tmp/dvc_local_storage`. The configuration is stored in [`.dvc/config`](.dvc/config).

```
Remote: localstorage → /tmp/dvc_local_storage (default)
```

### Data Versions

| Version | File | Description | Size |
|---------|------|-------------|------|
| **v1 (Raw)** | `data/MachineLearningRating_v3.txt` | Original pipe-delimited dataset with 1,000,098 rows × 52 columns | ~529 MB |
| **v2 (Cleaned)** | `data/cleaned_insurance_data.csv` | Cleaned CSV after data processing | ~489 MB |

### Cleaning Steps (v1 → v2)

The cleaning script ([`src/data_cleaning.py`](src/data_cleaning.py)) performs:

1. **Whitespace stripping** on all string columns
2. **Datetime conversion** for `TransactionMonth`
3. **Numeric type coercion** for columns like `TotalPremium`, `TotalClaims`, etc.
4. **Missing value handling**:
   - `CrossBorder`, `NumberOfVehiclesInFleet` → filled with `0`
   - `NewVehicle`, `WrittenOff`, `Rebuilt`, `Converted` → filled with `"Unknown"`
5. **Standardization** of `"Not specified"` → `NaN` for `Gender`, `MaritalStatus`, `Citizenship`, `Title`
6. **Drop empty columns** (if any are entirely `NaN`)
7. **Deduplication** of identical rows

### Reproducing the Data Pipeline

```bash
# 1. Clone the repository
git clone <repo-url>
cd insurance-risk-analytics

# 2. Install dependencies
pip install -r requirements.txt

# 3. Pull data from the DVC remote
dvc pull

# 4. (Optional) Re-run the cleaning pipeline
dvc repro
```

#### Key DVC Commands

| Command | Description |
|---------|-------------|
| `dvc pull` | Download tracked data files from the remote storage |
| `dvc push` | Upload tracked data files to the remote storage |
| `dvc repro` | Reproduce the pipeline (re-run cleaning if inputs changed) |
| `dvc status` | Check if data or pipeline stages are up to date |
| `dvc diff` | Show changes between commits for DVC-tracked files |

### DVC-Tracked Files

- `data/MachineLearningRating_v3.txt.dvc` — Pointer file for the raw dataset
- `data/cleaned_insurance_data.csv.dvc` — Pointer file for the cleaned dataset
- `dvc.yaml` — Pipeline definition (cleaning stage)
- `dvc.lock` — Lock file with exact hashes for reproducibility (generated by `dvc repro`)

### Changing the Remote Storage

To point to a different storage location:

```bash
# Local directory
dvc remote modify localstorage url /new/path/to/storage

# Or add a new remote (e.g., S3, GCS, Azure)
dvc remote add -d myremote s3://my-bucket/dvc-storage
```
